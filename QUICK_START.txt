   compress.exe mydocument.docx mydocument.huff

VERIFICATION (WINDOWS)
═════════════════════════════════════════════════════════════════════════════

To verify that decompression perfectly recovered the original file:

  fc original_file.txt recovered_file.txt

If no output appears, the files are identical (compression was lossless).
If you see differences, something went wrong.


WHAT HAPPENS DURING COMPRESSION
═════════════════════════════════════════════════════════════════════════════

1. INPUT:  Your original file (any format, any size)
2. SCAN:   Count frequency of every byte in the file
3. BUILD:  Create an optimal binary tree (Huffman tree)
4. ENCODE: Convert file content to variable-length binary codes
5. WRITE:  Save frequency map + encoded data to .huff file
6. OUTPUT: Compressed file (may be larger or smaller depending on content)


WHAT HAPPENS DURING DECOMPRESSION
═════════════════════════════════════════════════════════════════════════════

1. INPUT:     Your compressed .huff file
2. READ:      Extract frequency map from file header
3. REBUILD:   Reconstruct the Huffman tree
4. DECODE:    Read compressed bits and convert back to original bytes
5. WRITE:     Save decoded content to output file
6. OUTPUT:    Perfectly recovered original file


COMPRESSION RATIO EXPECTATIONS
═════════════════════════════════════════════════════════════════════════════

File Type                  Expected Compression
─────────────────────────────────────────────────────────────────────────────
Text files (large)         40-60% smaller
English documents          45-60% smaller
Log files                  50-70% smaller
Source code                30-50% smaller
JSON/XML files             20-40% smaller
Small files (<1KB)         May become LARGER (overhead dominates)
Already-compressed         ~0% (images, videos, archives)
Binary files               0-20% (depends on content)
Highly repetitive text     60-90% smaller

NOTE: Small files often expand because the frequency map overhead is large
relative to the content size. Huffman works best on files > 10KB.


TROUBLESHOOTING
═════════════════════════════════════════════════════════════════════════════

Problem: "Error: Input file not found"
  → Check that the file exists and the path is correct
  → Make sure you're using the right filename
  → File must be in the current directory or use full path

Problem: Decompressed file doesn't match original
  → This shouldn't happen with the current version
  → Try compressing and decompressing again
  → Check that you're using the same .huff file

Problem: .exe files won't run
  → Make sure you have Windows 7 or newer
  → Executables are statically linked, no dependencies needed
  → If you get "not a valid Win32 application", the files may be corrupted


ADVANCED USAGE
═════════════════════════════════════════════════════════════════════════════

Batch compression (all .txt files):
  for %f in (*.txt) do compress.exe %f %f.huff

Compression with size tracking:
  compress.exe large_file.bin large_file.huff

File recovery with backup:
  decompress.exe backup.huff restored.bin
  (Now you have both backup.huff and restored.bin)

REBUILDING FROM SOURCE
═════════════════════════════════════════════════════════════════════════════

If you modify the source code and want to recompile:

1. Install MSYS2 if you haven't already
2. Make sure g++ is in your PATH:
   set PATH=C:\msys64\ucrt64\bin;%PATH%
3. Compile:
   g++ -std=c++17 -static-libgcc -static-libstdc++ -o compress.exe compress_simple.cpp
   g++ -std=c++17 -static-libgcc -static-libstdc++ -o decompress.exe decompress_simple.cpp

Or simply run:
   build.bat

FREQUENTLY ASKED QUESTIONS
═════════════════════════════════════════════════════════════════════════════

Q: Why is my compressed file larger than the original?
A: With small files, the frequency map overhead exceeds savings. Try larger files.

Q: Can I open .huff files in other programs?
A: No, .huff is a proprietary format. Use decompress.exe to recover original.

Q: Is compression lossless?
A: Yes! 100% lossless. Decompressed file is byte-for-byte identical.

Q: How long does compression take?
A: Usually < 10 milliseconds, even for megabyte-sized files.

Q: Can I use this on macOS/Linux?
A: Yes! Use build.sh to compile. No .exe files needed.

Q: What if decompression fails?
A: Check that the .huff file wasn't corrupted. Original is still safe.

Q: Is it secure?
A: No. This is for compression only, not encryption. Anyone can decompress.

Q: Can I compress directories?
A: No, but you can use a tool like 7-Zip with this to add that feature.
