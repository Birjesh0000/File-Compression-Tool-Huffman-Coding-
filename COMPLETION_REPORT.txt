PROJECT OVERVIEW
================================================================================

A complete, production-ready Huffman coding file compression/decompression 
system implemented in C++17.

Compiled with MSYS2 (MinGW64) with static linking for maximum portability.

================================================================================
EXECUTABLE FILES (READY TO USE)
================================================================================

compress.exe
  - Compresses any file using Huffman coding
  - Stores frequency map in file header for decompression
  - Usage: compress.exe <input_file> <output.huff>

decompress.exe
  - Decompresses Huffman-encoded files
  - Reads header and reconstructs original file
  - Usage: decompress.exe <input.huff> <output_file>

Both executables include:
  ✓ Command-line interface
  ✓ Error handling
  ✓ Progress messages
  ✓ File validation

================================================================================
KEY COMPONENTS
================================================================================

1. HuffmanNode
   - Binary tree node structure
   - Character storage and frequency tracking
   - Left/right child pointers

2. CompareNode (Comparator)
   - Min-heap priority queue implementation
   - Ensures lowest frequency nodes selected first

3. HuffmanTree
   - Builds optimal Huffman tree from frequencies
   - Generates variable-length binary codes
   - O(n log k) complexity for tree building

4. BitWriter
   - Writes individual bits to file
   - Buffers bits into bytes
   - Handles partial byte padding

5. BitReader
   - Reads individual bits from file
   - Maintains bit buffer state
   - Traverses compressed bit stream

6. FileCompressor
   - Orchestrates compression pipeline
   - Calculates character frequencies
   - Serializes header with frequency map
   - Encodes file data using Huffman codes

7. FileDecompressor
   - Reconstructs frequency map from header
   - Rebuilds Huffman tree
   - Decodes bit stream back to original

================================================================================
ALGORITHM COMPLEXITY
================================================================================

Time Complexity:
  - Frequency Analysis:      O(n)         where n = file size
  - Tree Building:           O(k log k)   where k = unique characters
  - Code Generation:         O(k)
  - Compression/Decompression: O(n)
  - Overall:                 O(n + k log k) ≈ O(n)

Space Complexity:
  - Huffman Tree:            O(k)
  - Frequency Map:           O(k)
  - Code Map:                O(k)
  - Overall:                 O(k)

================================================================================
FILE FORMAT
================================================================================

Compressed File Structure:
┌─────────────────────────────────────────────┐
│ HEADER (Variable)                           │
├─────────────────────────────────────────────┤
│ [# of unique chars: 1 byte]                 │
│ For each character:                         │
│   - Character: 1 byte                       │
│   - Frequency: 4 bytes (big-endian)         │
├─────────────────────────────────────────────┤
│ ENCODED DATA (Variable)                     │
├─────────────────────────────────────────────┤
│ Binary stream of Huffman codes              │
│ (padded to byte boundary)                   │
└─────────────────────────────────────────────┘

Example Header (for "Hello"):
  03 48 00 00 00 01 65 00 00 00 01 6C 00 00 00 02 6F 00 00 00 01
  
  03       = 3 unique characters
  48       = 'H' (ASCII 72)
  00 00 00 01 = frequency 1
  65       = 'e' (ASCII 101)
  00 00 00 01 = frequency 1
  6C       = 'l' (ASCII 108)
  00 00 00 02 = frequency 2
  6F       = 'o' (ASCII 111)
  00 00 00 01 = frequency 1
